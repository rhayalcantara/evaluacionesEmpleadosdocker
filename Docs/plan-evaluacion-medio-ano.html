Bug #3 — Cálculo de puntuación ejecutado en evaluación cualitativa
src/app/Controllers/Evaluacion.ts · método grabar(supervisor)
El método grabar() llama a CalculoCompetencias() y calculaelpromediodesempeno(). Para medio año, las competencias son texto (observaciones), no números. Los cálculos producirán 0 y sobreescribirán los campos de puntuación con basura. Se necesita un flag que salte los cálculos numéricos cuando el periodo es de tipo medio_ano.
Fix sugerido en grabar() 
async grabar(supervisor: boolean): Promise<void> {
  // Verificar si el periodo es medio año para saltar cálculos
  const esMedioAno = this.model.periodId > 0
    && /* pasar el tipo del periodo o verificar el estado */;

  if (!esMedioAno) {
    // Solo calcular puntuaciones para evaluación final
    await this.calculaelpromediodesempeno(supervisor, ...);
    await this.CalculoCompetencias(supervisor);
  }
  // Guardar en todos los casos
}

Mejora #2 — Endpoint diferenciado por tipo de periodo
src/app/Controllers/Periodos.ts · método GetActivo()
Si coexisten dos periodos activos (uno final y uno medio año), GetActivo() retorna solo uno (el que devuelva el API). Se necesita GetActivoPorTipo(tipo: string) para que el sistema cargue el periodo correcto según el contexto del usuario.
Sugerencia 
public GetActivoPorTipo(tipo: string): Observable<IPeriodo_Dts> {
  return this.datos.getbyid<IPeriodo_Dts>(
    this.rutaapi + `/activo/${tipo}`
  );
}
